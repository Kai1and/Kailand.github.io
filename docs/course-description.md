# Курсовой проект

Тема: веб-информационная система бронирования оборудования с ролевым доступом и защитой данных.

## Назначение

Система предназначена для публикации объявлений об аренде оборудования, поиска нужной техники, общения арендатора с владельцем и оформления заявки на бронирование на выбранный период.

## Основные роли

- Гость: просматривает каталог, карточки оборудования и публичные профили.
- Пользователь: регистрируется, входит в систему, пишет владельцам, создает заявки на аренду, публикует свои объявления.
- Владелец: получает сообщения и заявки по своим объявлениям.
- Администратор: управляет пользователями, категориями, статусами заявок и справочниками.

## Основные функции

- Регистрация и вход по JWT.
- Каталог оборудования с карточками в стиле marketplace.
- Детальная страница товара с фото, ценой, описанием, городом и профилем владельца.
- Кнопка `Написать продавцу`, создающая защищенный диалог между арендатором и владельцем.
- Кнопка `Арендовать`, открывающая форму выбора периода аренды.
- Личный кабинет с бронированиями, объявлениями и сообщениями.
- Публичные профили пользователей без раскрытия пароля и служебных данных.
- Административные страницы пользователей и категорий.

## Защита данных

- Пароли хранятся только в виде bcrypt-хеша.
- Авторизация выполняется через JWT access token.
- Закрытые маршруты frontend защищены `ProtectedRoute`.
- Backend проверяет JWT для бронирований, профиля, чатов и административных операций.
- Сообщения доступны только участникам диалога: арендатору и владельцу оборудования.
- Публичный профиль возвращает только безопасные поля пользователя.
